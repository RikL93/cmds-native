1.2 Webview ↔ Native bridge contract
De webviewer roept deze bridge methodes aan via window.CMDSNative (zie src/lib/cmdsNative.ts in Lovable). Verifieer dat de Android WebAppInterface (of plugin) exact deze methodes implementeert met deze signatures:

Methode	Argument	Wanneer aangeroepen door web
onUnitLinked(unitId: string, callSign: string, eventId: string, organizationId: string)	4 strings	Bij selecteren eenheid in UnitView
onUnitUnlinked()	geen	Bij vrijgeven eenheid / uitloggen
onAuthChanged(accessToken: string, refreshToken: string, userId: string)	3 strings	Bij login/refresh
onAuthCleared()	geen	Bij logout
getNativeStatus()	geen → returns JSON string	Door diagnose-pagina
Belangrijk:

unitId is een UUID (geen call sign). Sla op in AsyncStorage / SharedPreferences onder key cmds_active_unit_id.
Sla óók cmds_active_event_id en cmds_active_organization_id op — nodig voor isolatie.
Bij onUnitUnlinked() / onAuthCleared() → wis alle keys en stop background GPS service.

Verifieer:

❗ Skip de POST als cmds_active_unit_id leeg is (en log de reden).

Gebruik Screen Wake Lock equivalent op native (Foreground Service met FOREGROUND_SERVICE_LOCATION).
AndroidManifest permissions aanwezig: ACCESS_FINE_LOCATION, ACCESS_BACKGROUND_LOCATION, FOREGROUND_SERVICE, FOREGROUND_SERVICE_LOCATION, POST_NOTIFICATIONS, WAKE_LOCK.

Maak een tweede diagnose-scherm naast de bestaande GPS diagnose. Bereikbaar via dezelfde menu/knop. Toon realtime (auto-refresh elke 2s) onderstaande velden:

=== UNIT LINK DIAGNOSE ===

[BRIDGE STATUS]

"Vraag webview om re-sync" knop → roept JS aan in de webview:


webView.evaluateJavascript("window.CMDSNative_requestResync && window.CMDSNative_requestResync()", null);
Implementeer aan webkant later een listener die opnieuw onUnitLinked triggert met de huidige selectie.

"Forceer GPS Push nu" → voert direct één POST uit en toont response in het diagnose-scherm.

"Kopieer diagnose naar klembord" → kopieert volledige output als platte tekst zodat ik die kan doorsturen.
